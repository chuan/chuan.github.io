Development Environment Setup For Hadoop 1.0 on Windows

Java Installation

Download JDK installer for Windows at the following address:

http://www.oracle.com/technetwork/java/javase/downloads/index.html

(Use JDK 1.6 is recommended for Hadoop 1.0 release.)

Ant Instllation

Download Apache Ant binary distribution in .zip format at the following address:

http://archive.apache.org/dist/ant/binaries/

(Ant 1.7 is recommended for Hadoop 1.0 build.)

Extract the zip to the location where you want Ant to be installed. For this instruction, the following location is used:

C:\Java

After this, Ant binaries should be available at:

C:\Java\apache-ant-[version]\bin

Note: you may run into some bugs if the installation path contains spaces, e.g. the default directory 'Program Files'.

Cygwin Installation (Optional)

Download and run the latest Cygwin installer at:

http://cygwin.com/setup.exe

Set PATH

Add a JAVA_HOME system environment variable with following value:

C:\Java\jdk1.6.0_32

Add an ANT_HOME system environment variable with the following value:

C:\Java\apache-ant-1.7.1

Add a PATH user environment variable with the following value:

%PATH%;%JAVA_HOME%\bin;%ANT_HOME%\bin;%USERPROFILE%\bin

Copy necessary Unix tools to %USERPROFILE%\bin. The following tools are used for now:

  • sh
  • sed
  • echo
  • gzip
  • tar

They are available from Cygwin installation at previous step. Other options may include MinGw, GnuWin32, etc.

Build

After path is set, we can build Hadoop Common from "Visual Studio Command Prompt" with the following command.

ant -Dwindows=true clean ms-package

Eclipse

Dowload an Eclipse package from the website. Classic is used in this guide.

http://www.eclipse.org/

Extract the package to a location. The following location is used for this guide.

C:\Java

Generate Eclipse config files using 'eclipse' task, i.e.

ant eclipse

Open Eclipse. Set the workspace directory to the parent folder of hadoop-common folder. For example, we can have hadoop-common code in folder 'core', and 'workspace' folder contains 'core'. In this case, we need to set Eclipse workspace to 'workspace'.

Create a new Java project via Eclipse menu: File - New - Java Project

Enter project name as the hadoop-common folder name, e.g. 'core'. Click next.

In the 'Source' tab windows, browse to: 'src\webapps'. Right click and select 'Use as Source Folder'.

In the 'Libraries' tab, click 'Add External Jars...'. Add tools.jar from JDK as an external Jar; 'tools.jar' can be found at JAVA_HOME\lib.

Open menu: Window - Preferences. In 'Java - Build Path - Classpath Variables', create a new variable 'ANT_HOME' with value of Ant installation location in the previous step.

The Eclipse environment is set up now.
